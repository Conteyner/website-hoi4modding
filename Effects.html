    Эффекты (также известные как команды) используются для динамического воздействия на игру в определенных рамках.
    Они представляют собой одноразовое изменение текущего состояния игры, не обладая способностью оказывать длительный эффект. Вместо этого модификаторы используются для оказания постоянного, непрекращающегося эффекта на состояние игры, который может быть представлен числом. Блоки эффектов не могут использоваться для прямого применения модификаторов, однако они могут добавлять что-то, что может применять модификаторы, чаще всего с помощью add_ideas.

    Эффекты используются на протяжении всей игры в различных сферах, чаще всего в национальных фокусах, событиях и решениях.

    Обратите внимание, что некоторые эффекты могут принимать значение из переменной, т.е. add_manpower = var:my_var Это отмечается <переменной> в параметрах эффекта. Информацию о переменных эффектов.

    Список эффектов может быть устаревшим. Полный, но несортированный список эффектов можно найти в /Hearts of Iron IV/documentation/effects_documentation.html или /Hearts of Iron IV/documentation/effects_documentation.md.

    Двойные диапазоны

    Следующие диапазоны могут быть использованы как диапазоны эффектов или триггеров; некоторые из них также могут быть использованы как правая сторона некоторых эффектов и триггеров в качестве цели. Если использование в качестве цели возможно, это отмечено в таблице. Некоторые двойные диапазоны могут иметь область видимости, которая меняется в зависимости от места использования, например, переменные, которые могут быть установлены на что угодно.

    Недопустимая цель события

    Что касается некоторых двойных диапазонов, то при их использовании может возникнуть ошибка "Invalid event target", как в common/national_focus/generic.txt:690: controller: invalid event target: controller, в то время как используемый диапазон не обязательно является целью события. Это означает, что область видимости не имеет определенной цели в контексте, в котором она используется, т.е. невозможно выбрать какую-либо одну цель при ее использовании. В случае controller = { ... }, как в примере, это означает, что область видимости проверяется или выполняется в состоянии, которое не контролируется ни одной страной. Такие состояния довольно нестабильны и могут легко привести к сбоям (например, при оценке воздушной миссии AI или при выполнении практически любого воздействия на них), поэтому эта ошибка никогда не должна возникать. На практике это полностью пропускается при оценке эффектов или триггеров: ни один из эффектов не будет выполнен; в качестве триггера не будет получено ни true, ни false. Однако, поскольку это может проверяться каждый тик, оставление этого как есть может привести к загромождению журнала ошибок. Чтобы избежать этого, можно использовать оператор if в эффектах или триггерах таким образом, чтобы двойная область видимости возникала только при необходимости, например, проверяя, что страна действительно является субъектом, перед проверкой

    		 Воспроизводит указанный звук один раз только для текущей страны. Звуковой эффект должен быть правильно определен в /Hearts of Iron IV/sound/. 1.6
    scoped_play_song <название песни из .asset>
    Музыкальный файл, расположенный в папке music и .asset 
    scoped_play_song = "general_peace_1"
    		 Воспроизводит аудиодорожку только для указанной страны. Чтобы песня работала, она должна быть определена на музыкальной станции. Более подробную информацию можно найти на странице Музыкальный моддинг. Если вы хотите просто воспроизвести звук, вместо этого следует использовать эффект scoped_sound_effect. 1.9.3

    		 Сохраняет динамику страны, гарантируя, что она не будет переработана для гражданской войны, даже если ее не существует. Обычно используется в сочетании с create_dynamic_country. 1.9
    force_update_map_mode limit = { ... }
    Триггеры, необходимые для обновления режима карты. Дополнительно.

    mapmode = <id>
    Идентификатор пользовательского режима карты. 

    force_update_map_mode = {
    limit = {
        is_ai = no
    }
    mapmode = my_map_mode
}
    		Принудительно обновляет указанный режим карты для игрока, вместо того чтобы ждать ежедневного обновления. Режимы карты определены в /Hearts of Iron IV/common/map_modes/*.txt 1.11

    create_dynamic_country original_tag = <tag>
    Оригинальный тег, который будет использоваться страной.

    copy_tag = <tag>
    Если указано, копирует материал из этого тега, а не из исходного тега.
    <effects>
    Эффекты, которые будут выполнены для новой динамической страны.

    		 
    create_dynamic_country = {
    original_tag = POL
    copy_tag = SOV
    add_political_power = 100
    transfer_state = 123
}
    		Создает новую динамичную страну, подобную тем, которые используются в гражданских войнах. 1.9
