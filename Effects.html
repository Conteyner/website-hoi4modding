    Эффекты (также известные как команды) используются для динамического воздействия на игру в определенных рамках.
    Они представляют собой одноразовое изменение текущего состояния игры, не обладая способностью оказывать длительный эффект. Вместо этого модификаторы используются для оказания постоянного, непрекращающегося эффекта на состояние игры, который может быть представлен числом. Блоки эффектов не могут использоваться для прямого применения модификаторов, однако они могут добавлять что-то, что может применять модификаторы, чаще всего с помощью add_ideas.

    Эффекты используются на протяжении всей игры в различных сферах, чаще всего в национальных фокусах, событиях и решениях.

    Обратите внимание, что некоторые эффекты могут принимать значение из переменной, т.е. add_manpower = var:my_var Это отмечается <переменной> в параметрах эффекта. Информацию о переменных эффектов.

    Список эффектов может быть устаревшим. Полный, но несортированный список эффектов можно найти в /Hearts of Iron IV/documentation/effects_documentation.html или /Hearts of Iron IV/documentation/effects_documentation.md.

    Недопустимая цель события

    Что касается некоторых двойных диапазонов, то при их использовании может возникнуть ошибка "Invalid event target", как в common/national_focus/generic.txt:690: controller: invalid event target: controller, в то время как используемый диапазон не обязательно является целью события. Это означает, что область видимости не имеет определенной цели в контексте, в котором она используется, т.е. невозможно выбрать какую-либо одну цель при ее использовании. В случае controller = { ... }, как в примере, это означает, что область видимости проверяется или выполняется в состоянии, которое не контролируется ни одной страной. Такие состояния довольно нестабильны и могут легко привести к сбоям (например, при оценке воздушной миссии AI или при выполнении практически любого воздействия на них), поэтому эта ошибка никогда не должна возникать. На практике это полностью пропускается при оценке эффектов или триггеров: ни один из эффектов не будет выполнен; в качестве триггера не будет получено ни true, ни false. Однако, поскольку это может проверяться каждый тик, оставление этого как есть может привести к загромождению журнала ошибок. Чтобы избежать этого, можно использовать оператор if в эффектах или триггерах таким образом, чтобы двойная область видимости возникала только при необходимости, например, проверяя, что страна действительно является субъектом, перед проверкой

    Управление потоком

    Эти диапазоны используются в диапазонах эффектов для управления выполнением эффектов.

    Если утверждения

    Оператор if позволяет выполнять эффекты только в том случае, если выполняются определенные триггеры. Это делается с помощью блока if = { ... } блок. Он служит блоком эффектов с дополнительным аргументом limit = { ... }, который представляет собой блок триггеров, где каждый триггер должен быть выполнен для проверяемой области действия. Например, следующее действие добавит 10% StabilityStability к стране, для которой оно выполняется, если она имеет положительное значение Political PowerPolitical Power и стабильность ниже 90%: 

    if = {
    limit = {
        has_political_power > 0
        stability < 0.9
    }
    add_stability = 0.1
}

    Если лимит не соблюден, то ни один из эффектов внутри не будет выполнен. Если же оно соблюдено, то будет выполнен каждый из них. Если ограничение опущено, то по умолчанию оно всегда будет истинным. Опционально else_if = { ... } (при этом limit = { ... } действует аналогичным образом) и else = { ... } могут быть добавлены. Если начальное ограничение в пределах if = { ... } ложно, он переходит к следующему else_if = { ... }, проверяя предел там. Если предел там ложен, то он переходит к следующему, пока не достигнет конца или else = { ... }. }. Существует два варианта: вложенный и не вложенный. В первом случае else_if или else помещается непосредственно внутри предыдущего if или else_if, а во втором - сразу после него. В случае дублирования предпочтительнее использовать невложенные операторы if. Вот пример, в котором используются не вложенные операторы if: 

    if = {
    limit = {
        stability < 0.3 
    }
    add_stability = 0.3
}
else_if = {
    limit = {
        stability < 0.6
    }
    add_stability = 0.2
}
else = {
    add_stability = 0.1 
}

    Во всплывающей подсказке отображаются только те эффекты, которые будут выполнены. Эффекты внутри невыполненного оператора if (или else/else_if, который не читается из-за выполнения оператора if) будут скрыты от игрока, как и триггер. Чтобы игрок не запутался, можно использовать пользовательские всплывающие подсказки эффектов, чтобы сообщить игроку, что будет делать данный блок эффектов, например, если его использовать внутри else.

    Случайные эффекты

    Если вы хотите, чтобы эффект имел случайный шанс быть выполненным или чтобы в противном случае ничего не произошло, блок random = { ... } - это самый простой способ добиться этого: 

    random = {
    chance = 80
    add_stability = 0.4
    add_war_support = 0.3
}

    Это, в частности, будет иметь 80% шанс добавить 40% стабильности и 30% поддержки войны и, соответственно, 20% шанс ничего не сделать. Шанс здесь находится на шкале от 0 до 100. Если вы хотите, чтобы игра выбирала между блоками эффектов, вместо этого можно использовать random_list. Например, если вы хотите, чтобы эффект случайным образом давал игроку один из четырех бонусов, вы сделаете следующее: 

    random_list = {
    10 = {
        add_stability = 0.5
    }
    10 = {
        add_manpower = 10000
    }
    10 = {
        add_war_support = 0.5
    }
    10 = {
        army_experience = 100
    }
}

    Число - это не шанс, а вес для каждого варианта, поскольку они не обязательно должны складываться в 100 или любое другое число. Вариант с весом 20 имеет в два раза больше шансов быть выбранным, чем вариант с шансом 10, например. В целом, вероятность того, что вариант будет выбран, равна весу варианта, деленному на сумму всех весов. Также можно использовать модификаторы для влияния на вес каждого возможного случайного эффекта или использовать переменные в качестве шансов. 

    random_list = {
    30 = {
        modifier = {
            factor = 1.3
            has_country_flag = inward_perfect_flag
        }
        add_stability = 0.5
    }
    25 = {
        add_manpower = 10000
    }
    20 = {
        add_war_support = 0.5
    }
    my_variable = { 
        army_experience = 100
    }
}

    Если установлен флаг страны inward_perfect_flag, то он умножит вышеуказанный шанс 30 на 1.3, чтобы получить 39. Между тем, my_variable возьмет значение соответствующей переменной temp или переменной текущего диапазона в качестве веса варианта.

    Обратите внимание, что если вы хотите создать повторяющееся решение, включающее случайный список, то по умолчанию это решение будет выбирать один и тот же случайный результат каждый раз, когда оно срабатывает в игре. Вы можете изменить это поведение, включив в блок решения следующую строку:

    fixed_random_seed = no

    Это относится только к решениям. В других местах случайное семя по умолчанию не фиксировано, поэтому этот аргумент не нужно устанавливать в значение "нет".

    Работа с всплывающими подсказками

    Под "подсказкой" в данном случае подразумевается текст, показываемый игроку в игре, который объясняет, что блок эффектов меняет в игре, например, "+50 Political PowerPolitical Power". Существует 3 способа редактирования всплывающей подсказки в блоке эффектов: hidden_effect = { ... } используется для того, чтобы скрыть эффекты внутри блока, в результате чего их выполнение не будет показано игроку. effect_tooltip = { ... } используется для того, чтобы поместить эффекты во всплывающую подсказку, не выполняя их на самом деле. custom_effect_tooltip = my_localisation_key используется для того, чтобы поместить произвольный абзац текста в качестве эффекта, который будет выполнен. Например, в этом примере награды за завершение фокуса используются все три эффекта: 

    completion_reward = { 
    hidden_effect = {
        every_subject_country = { country_event = my_event.1 }
    }

    Метаэффекты

    Метаэффекты позволяют вам использовать нединамические эффекты (те, которые не принимают модификаторы и могут использовать только статические маркеры или постоянные значения) так, как если бы они принимали переменные. 

    add_equipment_to_stockpile = {
    type = infantry_equipment_2
    amount = eq_amount
}

    В приведенном выше эффекте количество добавляемого оборудования является динамическим и может быть задано с помощью переменной "eq_amount". Однако этот эффект не позволяет использовать переменную в качестве типа снаряжения. Вы не можете сохранить "infantry_equipment_2" в переменной и использовать ее здесь. Однако метаэффекты позволят вам использовать переменные и скриптовую локализацию внутри них, чтобы создавать эффекты как тексты и запускать их. Давайте заставим предыдущий эффект принимать тип снаряжения и уровень снаряжения как переменные, хранящиеся в "eq_type" и "eq_level". 

    set_variable = { eq_type = 1 } 
set_variable = { eq_amount = 10 } 
set_variable = { eq_level = 2 }

meta_effect = { 
    text = {
        add_equipment_to_stockpile = {
            type = [EQ_TYPE]_[EQ_LEVEL]
            amount = eq_amount
        }
    }
    EQ_LEVEL = "[?eq_level|.0]" 
    EQ_TYPE = "[This.GetEquipmentName]" 
}                
            
    defined_text = {
    name = GetEquipmentName
    text = {
        trigger = {
            check_variable = { eq_type = 0 }
        }
        localization_key = "infantry_equipment"
    }
    text = {
        trigger = {
            check_variable = { eq_type = 1 }
        }
        localization_key = "artillery_equipment"
    }
}

    Как вы видите, мы создали метаэффект, который принимает два аргумента. Эти аргументы будут использоваться вместо параметров [EQ_TYPE] и [EQ_LEVEL] внутри метаэффекта. EQ_LEVEL будет заменен на [?eq_level|.0], который является целым значением eq_level (в данном случае 2.000 становится 2). EQ_TYPE немного сложнее, он заменяется скриптовой локализацией. Эта скриптовая локализация будет проверять переменную eq_type и в зависимости от ее значения будет возвращать токен ключа для оборудования. Если значение равно 0, то будет возвращено "infantry_equipment". Если значение равно 1, то будет возвращено "artillery_equipment". В итоге [EQ_TYPE] заменяется на "artillery_equipment", а [EQ_LEVEL] заменяется на "2", и в итоге наш эффект будет выглядеть следующим образом: 

    add_equipment_to_stockpile = {
    type = artillery_equipment_2
    amount = eq_amount
}

    что даст вам 10 артиллерийского_оборудования_2. debug = yes может быть добавлен к метаэффектам. Это позволит вывести конечный эффект в game.log при выполнении эффекта и облегчит отладку.

    Скриптовые эффекты

    Сценарные эффекты служат аналогично функциям, поскольку их можно определить в файле /Hearts of Iron IV/common/scripted_effects/*.txt, а затем использовать в других местах в качестве сокращенной версии. Наряду с этим, игра позволяет создавать пользовательские консольные команды, которые являются скриптовыми эффектами. Заскриптованный эффект определяется просто как 

    scripted_effect_name = {
	< effects >
}

    Этот пример можно использовать в качестве эффекта в обычном коде как scripted_effect_name = yes. К заскриптованным эффектам можно обратиться в консоли, набрав e scripted_effect_name для их запуска. Чтобы создать пользовательскую консольную команду, имя скриптового эффекта должно начинаться с d_. Сама консольная команда не включает d_, поэтому команда d_test_command будет запущена в консоли как test_command. В пользовательских консольных командах страна, выполняющая команду, является FROM, а ROOT - это выбранная страна, штат или символ. Все, что вводится после консольной команды, разделенное пробелами, например test_command 123 321 GER, добавляется в временный массив 'args'. Пример сценарного эффекта, который передаст каждое государство, введенное в качестве аргумента, в страну, выполняющую консольную команду, следующий 

    d_transfer_states = {
	for_each_scope_loop = {
		array = args
		FROM = {
			transfer_state = PREV
		}
	}
}

    используется как transfer_states 123 321
